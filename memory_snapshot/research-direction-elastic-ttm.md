Research project (SNU ARClab, algorithm–HW co-design). Platform: GPU (compute-dense, HBM-poor) + CXL-PNM (capacity-dense, low compute, decent near-bank BW); goal = workload where data movement between the two is minimized and the two are complementary. Attention+KV on CXL-PNM is considered stale; want a NEW workload in the TTT / Linear Attention domain.

**Chosen direction: "Elastic Test-Time Memory" — a linear-attention/TTT recurrent state that is RESIZABLE at runtime**, hosted in CXL-PNM, so recall/accuracy scales on-demand with context/difficulty WITHOUT retraining. Core algorithmic mechanism: nested/Matryoshka-structured state (nested-dropout on the state dimension during training → pick width k per request at inference). Hot leading dims → GPU HBM (fast path); cold extended dims → CXL-PNM (capacity). Extension (phase 2): grow nested levels with context length → "infinite context absorbed by PNM capacity."

**Why PNM not GPU (the justification):** the recurrent state update S_t = d_t⊙S_{t-1} + k_t v_t^T (and TTT gradient update) has Operational Intensity ≈ 1 FLOP/byte, CONSTANT in state size. GPU balance point ~295 FLOP/B (H100) → state update uses ~0.3% of GPU compute (Pimba: 73.8% of latency at batch 128). Scaling the state keeps OI ~1 → the accuracy knob and the PNM-fit knob are the SAME knob. Chunk-internal matmul / dual-form (OI tens–hundreds) stays on GPU.

**Key distinction settled:** state CAPACITY (parameter shape: d, L_M, MLP dim) is fixed at TRAINING (tied to trained projection weights) — you can't just crank d at inference. State OCCUPANCY (context length, batch aggregate, test-time adaptation) IS runtime-variable. The novel direction makes effective capacity runtime-tunable on ONE model via nested state.

**Target architecture:** TTT/Titans-family test-time memory (NOT Mamba2 — Pimba already did the simple linear-attn/SSM recurrence for throughput). Beachhead = Gated DeltaNet (reproducible via fla, strong recall: NIAH 98.4%, delta rule = clean rank-1 update). Then scale to TTT-MLP / Titans neural memory.

**First de-risking experiment:** add nested-dropout on head state dim to Gated DeltaNet (~160M), train on recall corpus, measure MQAR/NIAH recall vs inference-time width k. Success = monotonic graceful recall-vs-k curve with small "Matryoshka tax" vs fixed-width baselines.

**Open risks:** Matryoshka tax size; who picks k (heuristic vs learned controller vs surprise signal); recurrent nesting stability (error compounding under repeated truncation).
